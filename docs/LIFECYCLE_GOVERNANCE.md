# Lifecycle Governance

## 1. Purpose

Lifecycle governance records what happens to modular building infrastructure objects after registration.

## 2. Lifecycle Events

Supported event types:

- Registered
- Installed
- Inspected
- Maintained
- Repaired
- Replaced
- Retired

## 3. Lifecycle Stages

Supported stages:

- Design
- Manufactured
- Delivered
- Installed
- Inspected
- Operational
- Maintenance
- Replaced
- Retired

## 4. Governance Requirements

- Installation should have evidence.
- Inspection should have accountable inspector role.
- Maintenance should preserve notes and next action.
- Repair should identify affected component or port.
- Replacement should link old and new object records in a future version.
- Retirement should preserve final evidence and reason.

## 5. Runtime Readiness

Runtime readiness means the object can be referenced by BuildingOS. It does not mean the object is automated, certified, or safe for operation without human review.
