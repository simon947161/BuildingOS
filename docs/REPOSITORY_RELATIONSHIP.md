# Repository Relationship

## Repository Role

`BuildingOS` is the infrastructure governance platform.

It exists to define and test procedure, evidence, review, governance and ledger logic for infrastructure projects.

## Position in the Wider System

```text
Simon / Personal Headquarters
        ↓
BuildingOS / Infrastructure Governance Platform
        ↓
Applications such as Wagga ClimateOS, PPP/BOT Governance, Modular Building Interface and Community Lighthouse
```

## Relationship to Other Repositories

- `Simon` is the personal headquarters and project index.
- `climateos-knowledge-garden` is the internal Obsidian account book for CRP, radar notes and project memory.
- `eco-agent-system` provides environmental decision-support and ClimateOS / CarbonOS / WaterOS style evidence logic.
- `project-runtime-initiative` is planned as an independent human-AI project coordination runtime. It should serve BuildingOS through standards or adapters, not be merged into BuildingOS.
- `wagga-climate-os` is a regional application under the BuildingOS ecosystem.

## Boundary

BuildingOS should not absorb every idea.

Keep BuildingOS focused on infrastructure governance:

- procedure
- evidence
- review
- governance
- ledger
- infrastructure project intelligence

Do not use BuildingOS as the main home for:

- personal portfolio materials
- raw Obsidian notes
- book publication source
- generic multi-agent economy runtime
- local-only application experiments unless they are framed as BuildingOS applications

## Operating Rule

BuildingOS is a platform, not a warehouse.

New ideas should first be classified in the Knowledge Garden or Simon Project Index. Only infrastructure-governance materials should enter this repository.
