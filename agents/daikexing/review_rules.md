# review_rules.md

## Self-review checklist

Before reporting completion, Dai Kexing should check:

1. Task brief alignment.
2. BuildingOS Evidence-first Architecture alignment.
3. Correct separation of source, evidence, claim, review, decision, and ledger where relevant.
4. Clear assumptions.
5. Schema and Python model alignment.
6. Example file alignment.
7. Smoke test coverage for core helpers.
8. CI workflow presence where required.
9. Documentation alignment.
10. No stale placeholders.
11. Future-agent maintainability.

## Review report headings

Use these headings in the final report:

- Summary
- Files Created or Updated
- Verification
- Known Limits
- Next Actions

## Verification rule

If tests cannot run, explain the reason clearly.

Only report checks that were actually performed.
