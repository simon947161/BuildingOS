# BuildingOS

**Reality Operating System for Infrastructure Governance**

**One governance framework. Many infrastructure applications.**

BuildingOS is an open infrastructure governance framework for connecting
project intelligence, procedures, evidence, expert review, ledgers, and
continuous decision support across the full lifecycle of infrastructure
projects.

It is designed to bridge planning, financing, construction, operation, and
long-term governance. BuildingOS does not replace engineers, planners,
consultants, or government procedures. It provides a structured governance
layer so infrastructure projects can be discovered, reviewed, verified,
delivered, and operated with better evidence.

## Core Idea

Traditional project delivery often separates planning, finance, design,
construction, operation, and audit into different documents, teams, and
timelines.

BuildingOS treats infrastructure governance as a continuous operating process:

```text
Policy
-> Procedure
-> Evidence
-> Review
-> Decision
-> Ledger
-> Continuous Governance
```

The goal is not only to manage tasks, but to manage the procedures, evidence,
and decisions that make infrastructure projects trustworthy.

## Architecture

```text
Infrastructure Projects
-> Infrastructure Project Intelligence (IPI)
-> Procedure Engine
-> Evidence Engine
-> Review Engine
-> Governance Engine
-> Infrastructure Ledger
-> BuildingOS Runtime
```

## Core Modules

### 1. Infrastructure Project Intelligence (IPI)

Discovers and organizes infrastructure project information from public sources,
government records, tender portals, and project announcements.

### 2. Procedure Engine

Defines project stages, milestones, required documents, evidence gates, and
decision points for PPP, BOT, EPC, and other infrastructure workflows.

### 3. Evidence Engine

Separates source documents, claim-level evidence, and project profiles so weak
evidence is never treated as confirmed fact.

### 4. Review Engine

Records expert review from engineering, legal, finance, environmental,
delivery, and governance perspectives.

### 5. Governance Engine

Connects evidence, procedures, and expert review into structured go / no-go
decisions.

### 6. Infrastructure Ledger

Maintains a traceable record of project procedures, evidence, decisions,
reviews, and lifecycle events.

## M1 Specification Foundation

BuildingOS M1 Specification Foundation is locally complete.

The repository preserves:

- BuildingOS glossary and specification foundation documents.
- Evidence, Claim, Identity, Review, Procedure, Lifecycle, Registered Object,
  Governance Ledger, and Module Contract standards.
- Freeze records and Codex engineering reviews.
- Engineering workflow and incremental review governance.
- Roadmap, Core architecture, and engineering plan records.

No runtime implementation, Python package, SQLite schema, API, MCP tool, or PRI
implementation is introduced by M1.

## Positioning

BuildingOS is not a conventional BIM platform, project management app, or
consulting report system.

It is a procedure-driven infrastructure governance framework.

```text
Workflow generates Evidence.
Evidence forms Ledger.
Ledger builds Trust.
Trust enables Investment.
Investment builds Infrastructure.
Infrastructure serves Civilization.
```
